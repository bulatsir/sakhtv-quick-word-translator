# SakhTV Quick Word Translator — дизайн

**Дата:** 2026-07-06
**Статус:** утверждён (ревью спеки — агентом, пользователь не читает JS)

## Цель

Chrome-расширение для sakh.tv: при наведении курсора на слово в субтитрах
показывать его перевод с английского на русский во всплывающей подсказке.
Аналог easysubs.cc, но только для sakh.tv и только с минимальной функцией
«навёл — увидел перевод».

## Контекст (результаты разведки sakh.tv)

- Плеер: **JW Player 8.39** (`window.jwplayer` доступен на странице).
- Субтитры рендерятся текстом в DOM: `.jw-captions > .jw-text-track-cue`.
- Страницы просмотра: `https://sakh.tv/watch/<show>/<season>/<episode>/<voice>/`.
- Наличие субтитров зависит от выбранной озвучки (у оригинальных дорожек
  они есть, у большинства дубляжей — нет).
- Требуется авторизация на сайте, но расширение работает поверх уже
  открытой пользователем страницы — самому расширению авторизация не нужна.

## Решения, принятые с пользователем

| Вопрос | Решение |
|---|---|
| Направление перевода | en → ru, фиксировано |
| Источник перевода | Бесплатный endpoint Google Translate (`translate.googleapis.com/translate_a/single?client=gtx`) |
| Скорость | Предперевод всех слов реплики при её появлении + постоянный кэш |
| Объём v1 | Только перевод слова по наведению. Без паузы, без перевода фраз, без сохранения слов |
| Распространение | Личное пользование, загрузка через «Load unpacked» |

## Архитектура

4 файла, Manifest V3, без сборщиков и зависимостей:

```
manifest.json   — конфигурация расширения
content.js      — работа с субтитрами на странице
background.js   — service worker: переводы и кэш
styles.css      — тултип и подсветка слова
```

### manifest.json

- `manifest_version: 3`.
- `content_scripts`: `content.js` + `styles.css`, `matches: ["https://sakh.tv/watch/*"]`,
  `run_at: document_idle`.
- `background.service_worker: background.js`.
- `permissions: ["storage"]`.
- `host_permissions: ["https://translate.googleapis.com/*"]` — fetch к Google
  выполняется из service worker'а, чтобы не упереться в CORS со страницы.

### content.js

Обязанности:

1. **Поиск контейнера субтитров.** Плеер инициализируется асинхронно, поэтому
   ждём появления `.jw-captions` (интервал-опрос раз в 500 мс, до 60 с; если
   не нашли — тихо выходим: на странице нет плеера или субтитры выключены.
   Контейнер `.jw-captions` создаётся плеером сразу, даже при выключенных
   субтитрах, поэтому наблюдатель вешается один раз).
2. **Наблюдение.** `MutationObserver` на `.jw-captions` (childList + subtree +
   characterData). При каждом изменении — обработка текущей реплики.
3. **Оборачивание слов.** Для каждого текстового узла реплики: разбить по
   пробелам, каждое слово обернуть в `<span class="qwt-word">`. Пунктуация
   (`"Ran!"` → слово `ran`, знаки остаются снаружи запроса) отрезается при
   формировании запроса на перевод, но остаётся видимой в тексте. Защита от
   зацикливания: собственные мутации помечаются (`dataset.qwtDone`), обработчик
   их пропускает.
4. **Предперевод.** Сразу после оборачивания — одно сообщение в
   `background.js` со списком всех слов реплики (нормализованных:
   нижний регистр, без пунктуации). Ответ не блокирует UI.
5. **Тултип.** Один переиспользуемый `<div class="qwt-tooltip">`, добавленный
   **внутрь контейнера плеера** (`.jwplayer`), а не в `body` — иначе он
   невидим в полноэкранном режиме. Делегированные `mouseover`/`mouseout`
   на контейнере субтитров (`mouseenter` не всплывает и для делегирования
   не годится):
   - over на `.qwt-word` → запросить перевод слова (из кэша это мгновенно),
     показать тултип над словом;
   - если перевод ещё не пришёл — показать «…», обновить по получении;
   - leave → скрыть тултип.
6. **Изоляция ошибок.** Весь код content.js обёрнут так, чтобы любое
   исключение логировалось в консоль с префиксом `[QWT]` и не ломало плеер.

### background.js

Обязанности:

1. Принимает сообщения `{type: "translate", words: [...]}` из content.js.
2. Кэш двухуровневый:
   - **в памяти** (`Map`) — на время жизни service worker'а;
   - **`chrome.storage.local`** — постоянный, переживает перезапуск браузера.
     Ключ — слово, значение — перевод. Лимит storage.local (10 МБ) при
     словарном запасе сериалов недостижим, очистка не нужна.
3. Для слов, которых нет в кэше, — запрос к
   `https://translate.googleapis.com/translate_a/single?client=gtx&sl=en&tl=ru&dt=t&q=<word>`.
   Слова одной реплики переводятся одним batched-запросом (через перевод
   строки-разделителя) либо параллельными запросами по одному —
   решение за реализацией, но не более одного запроса на слово.
4. Ошибки сети/HTTP: ответ `{word, translation: null}`; content.js покажет
   «…», повторный hover повторит запрос. Никаких бесконечных ретраев.
5. Отвечает на каждое сообщение картой `{word: translation}`.

### styles.css

- `.qwt-word:hover` — лёгкая подсветка (например, подчёркивание/фон), чтобы
  было видно, на что наведён курсор.
- `.qwt-tooltip` — тёмная плашка с белым текстом над словом, `position:
  absolute`, `z-index` выше контролов плеера, `pointer-events: none`
  (чтобы тултип не перехватывал мышь и не вызывал мерцание).

## Поток данных

```
JW Player вставляет реплику
  → MutationObserver (content.js)
  → оборачивание слов в span
  → message "translate" со всеми словами → background.js
  → кэш / Google Translate → ответ {слово: перевод}
  → пользователь наводит на слово
  → тултип с переводом из локальной карты content.js
```

## Тестирование

Ручное, на sakh.tv (автотестов в v1 нет — проект персональный, вся
сложность в интеграции с живым плеером):

1. Серия с оригинальной озвучкой и включёнными субтитрами: слова
   подсвечиваются, тултип показывает перевод, задержки не ощущаются.
2. Реплики сменяются — обёртка работает на каждой новой реплике.
3. Полноэкранный режим — тултип виден.
4. Перемотка, пауза, смена качества — расширение не ломает плеер.
5. Офлайн/ошибка сети — тултип показывает «…», плеер работает.
6. Страница без субтитров — расширение бездействует, ошибок в консоли нет.

## Явно за рамками v1

- Пауза видео при наведении (кандидат №1 на v2 — пара строк через
  `jwplayer().pause()`).
- Перевод всей фразы целиком.
- Сохранение слов в список «на выучить».
- Другие языковые пары и автоопределение языка.
- Публикация в Chrome Web Store.

## Риски

| Риск | Оценка |
|---|---|
| Неофициальный endpoint Google перестанет работать | Низкая вероятность (живёт годами, им пользуется easysubs и десятки расширений). Замена — одно место в background.js |
| sakh.tv сменит плеер или классы | Починка — правка селекторов в content.js |
| Субтитры конкретной дорожки вшиты в видео | Расширение бессильно; пользователь выбирает дорожку с текстовыми субтитрами |
